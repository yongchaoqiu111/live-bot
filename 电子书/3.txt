Prompt 模板一：话术结构化清洗
用途
把Whisper转出的原始话术文本，按直播话术类型进行分类和结构化提取。

Prompt 正文
markdown
# 角色
你是一位专业的直播带货话术分析师，精通抖音/快手/淘宝直播的成交话术结构。

# 任务
分析下面这段直播话术原文，按照指定维度进行结构化提取。

# 输入话术原文
{{transcript_text}}

# 话术类型定义
1. 憋单型：制造稀缺感，如“只有XX单”“马上断货”“库存告急”
2. 逼单型：催促行动，如“倒计时”“手慢无”“最后XX秒”
3. 卖点型：产品功能介绍，如材质、功效、使用方法
4. 互动型：引导弹幕互动，如“要的打1”“觉得好的扣666”
5. 信任型：背书、资质、售后保障，如“假一赔万”“7天无理由”
6. 过渡型：转款、闲聊、暖场

# 输出格式（JSON）
{
  "segments": [
    {
      "start_time": "相对秒数",
      "end_time": "相对秒数",
      "text": "原话术片段",
      "type": "话术类型",
      "core_selling_points": ["卖点1", "卖点2"],
      "urgency_level": "高/中/低",
      "action_command": "行动指令内容，如无则为null",
      "has_countdown": true/false,
      "has_stock_warning": true/false,
      "has_gift": true/false
    }
  ],
  "summary": {
    "total_segments": 数量,
    "type_distribution": {"憋单型": 数量, "逼单型": 数量, ...},
    "avg_urgency_level": "高/中/低"
  }
}

# 注意事项
- 保持原文完整性，不要改写话术内容
- 时间戳保留Whisper输出的原始值
- 如果一句话包含多种类型，选择最主要的那个
Prompt 模板二：弹幕清洗与需求提取
用途
把原始弹幕数据，清洗出用户核心痛点、情绪分布、高频问题。

Prompt 正文
markdown
# 角色
你是一位直播用户行为分析专家，擅长从弹幕中提取用户的真实需求和情绪信号。

# 任务
分析下面这批直播弹幕数据，输出用户需求分析报告。

# 输入弹幕数据
{{danmu_list}}
格式：每条为 [时间戳] 用户：内容

# 输出格式（JSON）
{
  "total_count": 弹幕总数,
  "time_range": "开始时间 - 结束时间",
  "高频词统计": {
    "价格相关": ["词", 次数],
    "产品相关": ["词", 次数],
    "物流相关": ["词", 次数],
    "售后相关": ["词", 次数],
    "行动相关": ["词", 次数]
  },
  "情绪分布": {
    "急切购买": 数量,
    "犹豫观望": 数量,
    "质疑/负面": 数量,
    "中性提问": 数量
  },
  "核心痛点": [
    {
      "痛点": "用户关心什么",
      "代表弹幕": ["示例1", "示例2"],
      "出现频次": 次数
    }
  ],
  "峰值时刻": [
    {
      "timestamp": "时间点",
      "event": "发生了什么",
      "danmu_burst": 弹幕数,
      "top_keywords": ["关键词"]
    }
  ],
  "常见问题": [
    {"question": "用户常问的问题", "frequency": 次数}
  ]
}

# 注意事项
- 价格相关：多少钱、贵不贵、优惠、折扣
- 产品相关：尺寸、颜色、材质、用法
- 物流相关：发货、快递、多久到
- 售后相关：退换、保修、真假
- 行动相关：链接、怎么买、上架
Prompt 模板三：销量归因与策略生成（核心）
用途
把话术、弹幕、商品、销量四类数据关联起来，输出可复制的归因结论和优化建议。

Prompt 正文
markdown
# 角色
你是一位顶级直播带货策略顾问，擅长通过数据分析找出成交的关键因素，并给出可复制的优化方案。

# 输入数据
下方提供了某场直播的完整数据切片，每个切片包含：
- 时间窗口
- 主播话术（类型+原文）
- 弹幕特征（关键词+情绪）
- 当前商品
- 销量增量

{{data_slices}}

# 任务
请完成以下分析：

## 任务1：高转化时刻归因
找出销量增量最高的3个时间窗口，对每个窗口回答：
1. 销量增量具体是多少
2. 主播说了什么类型的话术、具体内容
3. 弹幕反应是什么（关键词+情绪）
4. 当时讲解什么商品
5. 归因结论：为什么这个窗口卖得好

## 任务2：低效时刻诊断
找出销量增量最低的3个时间窗口，对每个窗口回答：
1. 问题是什么（销量几乎没动）
2. 主播当时在做什么
3. 弹幕反应如何
4. 诊断结论：问题出在哪里

## 任务3：可复制策略输出
基于以上分析，输出以下内容：

### 3.1 高转化话术模板（3条）
每条包含：
- 话术类型
- 适用场景
- 话术原文模板（可直接复制使用）
- 预期效果参考

### 3.2 直播节奏优化建议
- 当前节奏的问题点
- 建议调整后的节奏
- 具体时间分配建议

### 3.3 商品排品建议
- 各商品的效率排序
- 建议的讲解顺序
- 建议的讲解时长

### 3.4 一句话核心结论
用一句话总结：这场直播卖得好的核心原因是什么

# 输出格式
请使用Markdown格式输出，结构清晰，便于直接作为报告内容。

Prompt调用建议
维度	建议
模型选择	Prompt 1-2 用GPT-4o-mini（便宜+够用）；Prompt 3用GPT-4o或Claude 3.5（需要更强推理）
批量处理	话术分5-10分钟一段调用，每段约2000字
成本控制	一场3小时直播，Prompt 1调用约20次，Prompt 2调用1次，Prompt 3调用1次，总成本约3-8元
结果缓存	相同主播/相同商品的相似话术，可缓存分类结果避免重复调用
